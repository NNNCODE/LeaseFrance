# Feature Specs

Use this folder for design notes once a backlog item is too large or ambiguous to implement directly.

A feature spec should usually include:
- goal
- user workflow
- data model changes
- IPC or preload changes
- UI entry points
- document or PDF impact
- acceptance criteria
- open questions

Recommended file naming:
- `tenant-ledger.md`
- `deposit-management.md`
- `unpaid-rent-follow-up.md`

Suggested flow:
1. Roadmap decides priority.
2. Backlog defines an actionable item.
3. Feature spec clarifies the implementation plan.
4. Claude Code implements the scoped work and updates the backlog status.
